# CalPush 料金モデル設計書

## 概要

CalPushをフリープラン付きの有料SaaSとして展開するための料金モデル。
現在のOSS版（Google Apps Script セルフホスト）はそのまま維持し、
**マネージド版（ホスティング版）** を有料サービスとして提供する。

---

## ビジネスモデルの方針

### セルフホスト版（OSS） → 無料のまま維持
- MIT License で引き続き公開
- ユーザー自身が GAS にデプロイ
- コミュニティからの信頼・認知獲得チャネル

### マネージド版（CalPush Cloud） → フリーミアム SaaS
- サーバー側でホスティング（GAS からの脱却）
- Web ダッシュボードで簡単セットアップ
- LINE連携のOAuth化でトークン管理不要
- 追加機能を有料プランで提供

## 有料化のための追加機能（Pro / Business）

### 1. 複数カレンダー統合
- 仕事用・プライベートなど複数カレンダーを一元通知
- カレンダーごとに通知ルールを設定可能

### 2. 通知のカスタマイズ
- 通知テンプレートの編集（Flex Message対応）
- リマインダーのタイミング設定（5分/10分/30分/1時間前）
- 通知時刻の変更（朝のスケジュール通知）

### 3. 高度な仮予約管理
- 仮予約の有効期限設定
- 承認フロー（Business）
- 外部向け空き時間共有リンク

### 4. 分析・レポート
- 月次スケジュールサマリー
- メール通知の統計
- 予定の傾向分析

### 5. 外部連携（Business）
- Webhook によるイベント通知
- Zapier / Make 連携
- Google スプレッドシートへの自動エクスポート

## Free → Pro への転換ポイント

ユーザーが自然に有料プランを検討するタイミング:

1. **LINE通知が200件/月に到達** → 通知上限の解放
2. **2つ目のカレンダーを追加したい** → 複数カレンダー対応
3. **仮予約を頻繁に使う** → 無制限の仮予約管理
4. **Gmail転送先を4ドメイン以上に** → 無制限ドメイン
5. **通知のタイミングを変えたい** → カスタマイズ機能

## 技術的な実装方針

### Phase 1: 基盤構築（〜2ヶ月）
- ユーザー認証（Google OAuth）
- プラン管理・課金基盤（Stripe）
- GAS → クラウドサーバー移行（Cloud Functions or Railway）

### Phase 2: Free プランリリース（〜3ヶ月）
- マネージド版の無料提供開始
- 利用量トラッキング
- プランアップグレード導線

### Phase 3: Pro プランリリース（〜4ヶ月）
- Stripe 決済連携
- 有料機能の実装（複数カレンダー、カスタマイズ）
- アップグレード/ダウングレードフロー

### Phase 4: Business プランリリース（〜6ヶ月）
- チーム管理機能
- 承認フロー
- 外部連携（Webhook, Zapier）

## リスクと対策

| リスク | 対策 |
|--------|------|
| LINE API の料金改定 | 通知の効率化（バッチ送信）、代替チャネル（Slack等）の準備 |
| 無料ユーザーのサーバーコスト | 利用量制限の適切な設定、段階的スケーリング |
| OSS版で十分という認識 | マネージド版ならではの利便性（セットアップ不要、自動アップデート）を訴求 |
| 競合の参入 | LINE × 日本語 × 仮予約 のニッチ領域で先行者優位を確立 |
