# CalPush 料金モデル設計書

## 概要

CalPushをフリープラン付きの有料SaaSとして展開するための料金モデル。
現在のOSS版（Google Apps Script セルフホスト）はそのまま維持し、
**マネージド版（ホスティング版）** を有料サービスとして提供する。

---

## ビジネスモデルの方針

### セルフホスト版（OSS） → 無料のまま維持
- MIT License で引き続き公開
- ユーザー自身が GAS にデプロイ
- コミュニティからの信頼・認知獲得チャネル

### マネージド版（CalPush Cloud） → フリーミアム SaaS
- サーバー側でホスティング（GAS からの脱却）
- Web ダッシュボードで簡単セットアップ
- LINE連携のOAuth化でトークン管理不要
- 追加機能を有料プランで提供

## Google OAuth 連携によるゼロ設定オンボーディング

### 現行（GAS版）のセットアップ課題

現在のCalPush（GAS版）は、ユーザーに以下の手動作業を要求する：

1. Google Apps Script プロジェクトを作成
2. Code.js をコピー＆ペースト
3. LINE Developer Console で Messaging API チャネルを作成
4. Channel Access Token / User ID をスクリプトプロパティに設定
5. Calendar ID を設定
6. Gmail通知用のドメインを手動登録
7. トリガーを手動設定（初回実行）

→ 技術リテラシーの高いユーザーでも **15〜30分** かかる。

### CalPush Cloud の目標: 「3クリックで完了」

```
ユーザー体験:
  1. 「Googleでログイン」ボタン → Google OAuth同意画面 → 許可
  2. 「LINE公式アカウントを接続」→ Channel ID/Secret 入力（ウィザード）
  3. 通知設定を確認 → 完了
  → 所要時間: 約3分
```

### Google OAuth で取得するスコープと審査区分

#### CalPushに必要なスコープ一覧

| スコープ | 用途 | 分類 | 審査 |
|---------|------|------|------|
| `openid` | ユーザー認証（ログイン） | Non-sensitive | 不要 |
| `profile` | 表示名・アイコン取得 | Non-sensitive | 不要 |
| `email` | メールアドレス取得 | Non-sensitive | 不要 |
| `calendar.readonly` | カレンダー予定の読み取り | **Sensitive** | **OAuth検証（無料）** |
| `calendar.events` | 仮予約の作成・編集 | **Sensitive** | **OAuth検証（無料）** |
| `gmail.readonly` | Gmail未読メールの検索・読み取り | **Restricted** | **OAuth検証 + セキュリティ評価（有料）** |

#### 審査区分の影響

| 区分 | 審査内容 | 費用 | 期間 |
|------|---------|------|------|
| **Non-sensitive** | 審査不要 | 無料 | 即時 |
| **Sensitive** | Google OAuth検証（デモ動画・プライバシーポリシー提出） | **無料** | 数日〜数週間 |
| **Restricted** | OAuth検証 + CASA等セキュリティ評価 + 年次再審査 | **有料（数千〜数万ドル）** | 数週間〜数ヶ月 |

---

### Gmail スコープの戦略的判断

#### 問題: `gmail.readonly` は Restricted スコープ

Gmail APIのスコープはすべて **Restricted（制限付き）** に分類される。
これにより以下のハードルが発生する：

| ハードル | 詳細 |
|---------|------|
| セキュリティ評価（CASA） | 第三者監査機関による評価が必要。費用は数千〜数万ドル |
| 年次再審査 | 毎年セキュリティ評価を再受験する必要がある |
| データ取扱いの厳格な制約 | Gmail データのサーバー保存・転送に制限 |
| 審査期間 | 数週間〜数ヶ月、事業計画に大きな影響 |

#### 推奨: 段階的スコープ戦略

| フェーズ | スコープ | Gmail機能 | 審査負荷 |
|---------|---------|----------|---------|
| **Phase 1（MVP）** | `calendar.readonly` + `calendar.events` | **なし**（Gmail転送はOSS版のみ） | Sensitive のみ（**無料**） |
| **Phase 2（成長期）** | Phase 1 + Gmail通知の代替手段 | **Google Apps Script アドオン経由** | 追加審査不要 |
| **Phase 3（法人化後）** | Phase 1 + `gmail.readonly` | フル機能 | Restricted（**有料**） |

#### Phase 2 の Gmail 代替手段

Gmail の Restricted スコープを回避しつつ、メール通知機能を提供する方法：

| 方法 | 仕組み | メリット | デメリット |
|------|--------|---------|----------|
| **GAS アドオン連携** | ユーザーのGASがGmailを読み、CalPush CloudにWebhookで通知 | Restricted審査不要、ユーザーの既存権限を利用 | GASの設定が一部残る |
| **Gmail転送ルール** | Gmailフィルタで特定メールをCalPush受信用アドレスに転送 | 完全にAPI不要 | ユーザー側の設定が必要 |
| **Google Workspace Add-on** | Workspace Marketplace にアドオンとして公開 | 組織内は審査緩和 | Marketplace審査が別途必要 |

### 全体アーキテクチャ（CalPush Cloud）

```
[ユーザー]
   │
   ├── (1) 「Googleでログイン」
   │       → Google OAuth 2.0（openid + profile + email + calendar.readonly + calendar.events）
   │       → CalPush が refresh_token を安全に保存
   │
   ├── (2) 「LINE公式アカウントを接続」
   │       → Channel ID / Channel Secret 入力（ウィザード）
   │       → CalPush が Channel Access Token v2.1 を JWT で自動発行
   │       → Webhook URL を自動生成・表示
   │
   └── (3) 通知開始
           │
           ├── CalPush Server（定期実行）
           │     ├── Google Calendar API で予定取得（refresh_token 使用）
           │     ├── 朝のスケジュール通知 → ユーザーのLINE公式アカウント経由で Push API
           │     └── 5分前リマインダー → 同上
           │
           └── LINE Webhook 受信
                 └── ユーザーのコマンド → Reply API（無料）で応答
```
