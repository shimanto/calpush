# CalPush 料金モデル設計書

## 概要

CalPushをフリープラン付きの有料SaaSとして展開するための料金モデル。
現在のOSS版（Google Apps Script セルフホスト）はそのまま維持し、
**マネージド版（ホスティング版）** を有料サービスとして提供する。

---

## ビジネスモデルの方針

### セルフホスト版（OSS） → 無料のまま維持
- MIT License で引き続き公開
- ユーザー自身が GAS にデプロイ
- コミュニティからの信頼・認知獲得チャネル

### マネージド版（CalPush Cloud） → フリーミアム SaaS
- サーバー側でホスティング（GAS からの脱却）
- Web ダッシュボードで簡単セットアップ
- LINE連携のOAuth化でトークン管理不要
- 追加機能を有料プランで提供

## 有料化のための追加機能（Pro / Business）

### 1. 複数カレンダー統合
- 仕事用・プライベートなど複数カレンダーを一元通知
- カレンダーごとに通知ルールを設定可能

### 2. 通知のカスタマイズ
- 通知テンプレートの編集（Flex Message対応）
- リマインダーのタイミング設定（5分/10分/30分/1時間前）

### 3. 高度な仮予約管理
- 仮予約の有効期限設定
- 承認フロー（Business）
- 外部向け空き時間共有リンク

### 4. 分析・レポート
- 月次スケジュールサマリー
- メール通知の統計
- 予定の傾向分析

### 5. 外部連携（Business）
- Webhook によるイベント通知
- Zapier / Make 連携
- Google スプレッドシートへの自動エクスポート

### CalPushの通知パターンと課金区分

CalPushで使用しているLINE Messaging APIの呼び出しを分析した結果：

| API | 用途 | 課金区分 |
|-----|------|---------|
| **Push API** (`/v2/bot/message/push`) | 朝のスケジュール通知、5分前リマインダー、Gmail転送通知 | **課金対象** |
| **Reply API** (`/v2/bot/message/reply`) | LINEコマンドへの応答（今日の予定、予定追加、通知先一覧 等） | **無料（対象外）** |
| Mark as Read API | 既読処理 | 無料（対象外） |
| Quota Consumption API | 送信数確認 | 無料（対象外） |

**重要**: CalPushのPush通知（課金対象）は自動通知のみ。ユーザー操作による応答はReply APIで無料。

### アーキテクチャ別のLINE原価構造

#### パターンA: 1ユーザー = 1 LINE公式アカウント（セルフホスト型）

各ユーザーが自分のLINE公式アカウントを使う現行モデル。
CalPush Cloud でも、ユーザーが自分のLINEチャネルを接続する方式。

| 項目 | 内容 |
|------|------|
| LINE原価 | **¥0（CalPush側の負担なし）** |
| ユーザー側のLINE費用 | コミュニケーションプラン（無料・200通/月） |
| メリット | CalPush側のLINEコスト完全ゼロ |
| デメリット | ユーザーに LINE Developer 設定を求める必要がある |

#### パターンB: CalPush共有LINE公式アカウント（マネージド型）

CalPushが1つのLINE公式アカウントを運営し、全ユーザーにPush通知。

| ユーザー数 | 月間Push通数 | LINEプラン | 月額固定費 | 追加メッセージ料金 | **LINE原価合計** |
|-----------|------------|-----------|-----------|-------------------|-----------------|
| 100人 | 14,000通 | コミュニケーション→**ライト** | ¥5,000 | ¥0（5,000通内） | - |
| 100人 | 14,000通 | **スタンダード** | ¥15,000 | ¥0（30,000通内） | **¥15,000** |
| 500人 | 70,000通 | スタンダード | ¥15,000 | (30,000通無料) + 40,000通×¥3.0 = ¥120,000 | **¥135,000** |
| 1,000人 | 140,000通 | スタンダード | ¥15,000 | 50,000×¥3.0 + 50,000×¥2.8 + 10,000×¥2.6 = ¥316,000 | **¥331,000** |
| 5,000人 | 700,000通 | スタンダード | ¥15,000 | 50,000×¥3.0 + 50,000×¥2.8 + 100,000×¥2.6 + 470,000×要相談 | **¥1,000,000超** |

> **注**: 100人の場合、ライトプラン（¥5,000/月）では5,000通しか送れず不足するため、スタンダードプラン（¥15,000/月）が必要。

### 推奨アーキテクチャ: ハイブリッドモデル（パターンA + 付加価値）

| 方針 | 内容 |
|------|------|
| LINE通知の送信 | **ユーザー自身のLINE公式アカウント経由**（パターンA） |
| CalPush Cloudの価値 | セットアップ簡素化、Web UI、複数カレンダー、仮予約管理等 |
| LINE設定の簡素化 | OAuth連携でトークン入力不要にする（LINE Login Channel連携） |
| CalPush側のLINE原価 | **¥0** |

このモデルでは:
- CalPushはLINEメッセージ送信の原価を一切負担しない
- 各ユーザーのLINE無料枠（200通/月）で個人利用は十分カバー
- ヘビーユーザーはLINE側で自分でライト/スタンダードにアップグレード
- CalPushの有料価値は「機能」であり「通知インフラ」ではない

## 技術的な実装方針

### Phase 1: 基盤構築（〜2ヶ月）

- マネージド版の無料提供開始
- 利用量トラッキング
- プランアップグレード導線

### Phase 3: Pro プランリリース（〜4ヶ月）
- Stripe 決済連携
- 有料機能の実装（複数カレンダー、カスタマイズ）

### Phase 4: Business プランリリース（〜6ヶ月）
- チーム管理機能
- 承認フロー
- 外部連携（Webhook, Zapier）
